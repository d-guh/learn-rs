# Unrecoverable Errors with `panic!`
Panics happen when:
- Something in the code causes a panic (out of bounds array access)
- Calling the `panic!` macro directly

When a panic happens, Rust will either walk back up the stack "unwind" or Abort (if configured), which forces the OS to clean up memory.

Example panic:
```rs
fn main() {
    panic!("crash and burn");
}
```
You can add environment variable `RUST_BACKTRACE=1` to help debug panics

Here's another example originating from code and not the macro itself:
```rs
fn main() {
    let v = vec![1, 2, 3];

    v[99];
}
```
