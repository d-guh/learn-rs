# Storing UTF-8 Encoded Text with Strings
Strings are complicated. They are still structs and stored on the heap.
## Defining Strings
"string" can refer to `String` or `str` (usually seen as `&str`). `String` is growable, stored on heap, and is basically a `Vec<u8>`, while `str` is a slice of bytes.
## Creating a New String
Shares most operations with `Vec<T>`. New method:
```rs
    let mut s = String::new();
```
`to_string()` method:
```rs
    let data = "initial contents";

    let s = data.to_string();

    // The method also works on a literal directly:
    let s = "initial contents".to_string();
```
`from` method:
```rs
    let s = String::from("initial contents");
```
Note that strings are UTF-8 encoded:
```rs
    let hello = String::from("السلام عليكم");
    let hello = String::from("Dobrý den");
    let hello = String::from("Hello");
    let hello = String::from("שלום");
    let hello = String::from("नमस्ते");
    let hello = String::from("こんにちは");
    let hello = String::from("안녕하세요");
    let hello = String::from("你好");
    let hello = String::from("Olá");
    let hello = String::from("Здравствуйте");
    let hello = String::from("Hola");
```
## Updating a String
`String`s can grow in size and its contents can change just like `Vec<T>`.
### Appending with `push_str` or `push`
Grow a string with `push_str`:
```rs
    let mut s = String::from("foo");
    s.push_str("bar");
```
Note it uses a string slice to avoid taking ownership:
```rs
    let mut s1 = String::from("foo");
    let s2 = "bar";
    s1.push_str(s2);
    println!("s2 is {s2}");
```
`push` works for a single character:
```rs
    let mut s = String::from("lo");
    s.push('l');
```
### Concatenating with `+` or `format!`
You can combine strings as follows:
```rs
    let s1 = String::from("Hello, ");
    let s2 = String::from("world!");
    let s3 = s1 + &s2; // note s1 has been moved here and can no longer be used
```
This takes place using the `add` method. Note the `&` for the 2nd string, since we must add using reference per the function signature:
```rs
fn add(self, s: &str) -> String {
```
Adding can get unwieldy:
```rs
    let s1 = String::from("tic");
    let s2 = String::from("tac");
    let s3 = String::from("toe");

    let s = s1 + "-" + &s2 + "-" + &s3;  // String type
```
Use `format!`, and it doesn't take any ownership:
```rs
    let s1 = String::from("tic");
    let s2 = String::from("tac");
    let s3 = String::from("toe");

    let s = format!("{s1}-{s2}-{s3}");  // String type
```
## Indexing into Strings
Strings don't support traditional indexing:
```rs
    let s1 = String::from("hi");
    let h = s1[0];
```
### Internal Representation
Since `String` is a wrapper over `Vec<u8>`, the len/bytes do not correspond to characters:
```rs
    let hello = String::from("Hola");  // len = 4
    let hello = String::from("Здравствуйте");  // len = 24
```
### Bytes, Scalar Values, and Grapheme Clusters
Bytes = lowest level, u8 vec:
```rs
[224, 164, 168, 224, 164, 174, 224, 164, 184, 224, 165, 141, 224, 164, 164,
224, 165, 135]
```
Unicode scalar values:
```rs
['न', 'म', 'स', '्', 'त', 'े']
```
grapheme clusters:
```rs
["न", "म", "स्", "ते"]
```
## Slicing Strings
Be careful since slicing over bytes, if only part of a chars bytes are indexed, it will panic:
```rs
let hello = "Здравствуйте";

let s = &hello[0..4];  // "Зд"
```
`&hello[0..1]` would panic
## Iterating Over Strings
For loops chars:
```rs
for c in "Зд".chars() {
    println!("{c}");
}
//З
//д
```
For loops bytes:
```rs
for b in "Зд".bytes() {
    println!("{b}");
}
//208
//151
//208
//180
```
## Handling the Complexities of Strings
Strings are complicated. Use built in functionality for things like `contains` and `replace`
