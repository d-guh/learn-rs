# Storing Lists of Values with Vectors
`Vec<T>` is a vector, just like C++ vectors, stored on heap, used like a list.

## Creating a New Vector
You can create a new empty vector as follows (requires type annotation):
```rs
    let v: Vec<i32> = Vec::new();
```
And create a vector with values with the `vec!` macro:
```rs
    let v = vec![1, 2, 3];
```
Note type annotation not required as it inferred `i32`
## Updating a Vector
Push, similar to C++:
```rs
    let mut v = Vec::new();

    v.push(5);
    v.push(6);
    v.push(7);
    v.push(8);
```
## Reading Vectors
Via indexing or `get` method. Note that indexing returns value, get returns `Option<&T>`, which will not panic if no value is found.
```rs
    let v = vec![1, 2, 3, 4, 5];

    let third: &i32 = &v[2];
    println!("The third element is {third}");

    let third: Option<&i32> = v.get(2);
    match third {
        Some(third) => println!("The third element is {third}"),
        None => println!("There is no third element."),
    }
```
Note ownership rules still apply, can't hold immutable reference and mutate in same scope, the following code will not compile:
```rs
    let mut v = vec![1, 2, 3, 4, 5];

    let first = &v[0];

    v.push(6);

    println!("The first element is: {first}");
```
## Iterating Over the Values in a Vector
For loop method:
```rs
    let v = vec![100, 32, 57];
    for i in &v {
        println!("{i}");
    }
```
Mutable example (note `*` dereference)
```rs
    let mut v = vec![100, 32, 57];
    for i in &mut v {
        *i += 50;
    }
```
## Using an Enum to Store Multiple Types
Just like vectors of structs or similar types in C/C++:
```rs
    enum SpreadsheetCell {
        Int(i32),
        Float(f64),
        Text(String),
    }

    let row = vec![
        SpreadsheetCell::Int(3),
        SpreadsheetCell::Text(String::from("blue")),
        SpreadsheetCell::Float(10.12),
    ];
```
## Dropping a Vector Drops Its Elements
Like structs, a vector is freed when it goes out of scope:
```rs
    {
        let v = vec![1, 2, 3, 4];

        // do stuff with v
    } // <- v goes out of scope and is freed here
```
