# Packages and Crates
Crate: Smallest amount of code the compiler considers at a time.\
Crates contain modules.\
Crates have 2 forms:

- Binary Crate: Programs you can compile to an executable that can be run, must have `main`
- Library Crate: Don't have `main`, don't compile to an executable, define functionality to be shared between projects.

Package: Bundle of one or more crates that provides a set of functionality.\
Packages must contain only one crate. (1 max lib, unlimited bin)

Create a package:
```
$ cargo new my-project
     Created binary (application) `my-project` package
$ ls my-project
Cargo.toml
src
$ ls my-project/src
main.rs
```
`Cargo.toml` gives us a package, `src/main.rs` is the crate root of binary crates, `src/lib.rs` for library crates.

If a package contains both `src/main and src/lib`, it has 2 crates: a binary and library, both with the same name as the package.\
A package can have multiple binary crates by placing files in the `src/bin` directory, each file is a separate binary crate.
