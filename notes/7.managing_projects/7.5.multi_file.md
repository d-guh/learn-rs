# Separating Modules into Different Files
To extract modules into files rather than having everything in the crate root file, we can split the code into different files.

We can move the `front_of_house` module to its own file from the previous example. Move this modules code to a file named `src/front_of_house.rs`, and make `src/lib.rs` only contain:
```rs
mod front_of_house;

pub use crate::front_of_house::hosting;

pub fn eat_at_restaurant() {
    hosting::add_to_waitlist();
}
```
and `src/front_of_house.rs`:
```rs
pub mod hosting {
    pub fn add_to_waitlist() {}
}
```
Once you load a file using `mod`, you don't need to do it again for the rest of the project. You should then refer to the loaded files code using the path to where it was declared.

To start, in `src/font_of_house.rs`:
```rs
pub mod hosting;
```
Then, create `src/front_of_house` directory, and a `hosting.rs` file within this directory.
`src/front_of_house/hosting.rs`:
```rs
pub fn add_to_waitlist() {}
```
## Alternate File Paths
Note that the compiler supports the following paths:
```
src/front_of_house.rs (what we covered)
src/front_of_house/mod.rs (older style, still supported path)

src/front_of_house/hosting.rs (what we covered)
src/front_of_house/hosting/mod.rs (older style, still supported path)
```
